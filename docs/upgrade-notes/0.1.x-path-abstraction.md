# Upgrade Notes: Path Abstraction (Schema v37)

## What changed

Schema v37 introduces a root/relative path discipline. All file paths are
now stored relative to their watch folder root, not as absolute paths.
This enables future host/Docker deployment parity.

## What happens on first startup

1. The daemon detects the schema bump (v36 → v37).
2. **Phase 1-2 (instant):** SQLite ingest tables are truncated and
   `tracked_files` is rebuilt without the `file_path` column.
3. **Phase 2b (seconds):** Qdrant `projects`, `libraries`, and `images`
   collections are deleted and recreated. Rules and scratchpad are
   preserved.
4. **Phase 3 (minutes):** All watch folders are re-walked and files
   re-enqueued for ingestion with relative paths.
5. **Phase 4 (automatic):** When the queue drains, the migration marker
   is deleted.

## Expected re-ingest time

| Repo size | Approximate time |
|-----------|-----------------|
| < 10K files | < 5 minutes |
| 10K-50K files | 5-15 minutes |
| 50K-200K files | 15-45 minutes |
| > 200K files | 45+ minutes |

Times depend on disk speed, embedding model, and Qdrant throughput.

## Monitoring progress

```bash
wqm status health    # Shows migration banner with percentage
wqm queue stats      # Shows pending/in-progress queue items
```

The daemon logs progress at INFO level every 1000 files.

## Crash safety

The migration is crash-safe. If the daemon is killed at any point:
- The `relative_path_migration_in_progress` marker row persists.
- On next startup, the daemon resumes from Phase 2b (idempotent).
- No data corruption occurs; at worst, some files are re-processed.

## Breaking changes for downstream consumers

- `tracked_files.file_path` column no longer exists. Use
  `watch_folders.path || '/' || tracked_files.relative_path` to
  reconstruct absolute paths.
- `unified_queue.file_path` now stores relative paths, not absolute.
- gRPC handlers reject invalid paths with `InvalidArgument` status.
  Root paths must be absolute; content paths must be relative (no
  leading `/`, no `..` segments).

## Control port

Schema v37 coincides with the cross-process single-instance lock
(port 7799). If another memexd instance is running, the second
instance will refuse to start. Use `--control-port` to override.
